## Repos

- **web-platform** — Next.js app. UI + BFF (server-side API routes). The MMBR product.
- **ai-platform** — Python FastAPI backend. Source of truth for QAP behavior in MMBR.
- **qubika-agentic-platform** — Upstream QAP from `thisisqubika/`. Read-only reference for drift checks. **Never** source of truth for MMBR.
- **infraestructure-iac** — Terraform for dev/qa/prod. Manual `terraform apply`, no CI.

## Deployed services (ECS, per env: dev/qa/prod)

- **qbrick-\*** — the QAP backend in production. Same code as `ai-platform/`. No UI. Internal-only, reached via service discovery.
- **web-platform-\*** — the MMBR product app. Public-facing.
- **backend-ui-\*** — QAP admin UI (Next.js). Despite the name, it **is a frontend**. Manages workflows/agents inside `qbrick`.

## QAP / QBricks

- **QAP** — Qubika Agentic Platform. The framework. Acronym used in env vars and code.
- **QBricks** — same thing, used in prose and docs. ECS service is `qbrick-*` (lowercase, singular). Treat `QAP`, `QBricks`, and `qbrick` as one concept.
- **Workflow** — top-level QAP automation unit. Contains agents. Has its own API-key bundle.
- **Agent** — addressable AI participant inside a workflow. Identified by `QAP_AGENT_ID`.
- **Conversation / Message / Citation / Document** — QAP-owned entities. Persistence lives in `qbrick`'s Postgres, not in `web-platform`.

## Identity & tenancy

- **Auth0** — identity provider. Owns login, passwords, JWTs, sessions.
- **Plant** — MMBR multi-tenancy boundary. Users are assigned to one or more plants; data is scoped per plant.
- **Plant scoping** — the discipline of always filtering by `plantId` (or set of assigned plants) at request time.
- **Roles** — `operator` (chat with assigned plants), `admin` (plant manager, metrics scope TBD), `superadmin` (full access, QBricks management, Documents Panel). Defined in `types/domain.ts` as `UserRole`.
- **Source of truth for role** — `mmbr.users.role` in Postgres, **not** the Auth0 ID-token claim. The claim is informational; authorization always re-reads the DB.
- **Documents Panel** — superadmin-only feature in `web-platform`. Classic defense-in-depth example (UI hide + route guard + API guard, all DB-backed).

## Cross-cutting concepts

- **BFF** — Backend-for-Frontend. The Next.js API routes in `web-platform` that wrap `qbrick` for the browser.
- **Auth secret triangle** — same HMAC under three env-var names: `NEXTAUTH_SECRET` on `backend-ui` and `qbrick` (legacy alias), `QAP_JWT_SECRET` on `web-platform` (canonical). All three must hold the same value.
- **Per-workflow API bundle** — `QAP_API_KEY` + `QAP_API_SECRET` + `QAP_AGENT_ID`. Generated by `POST /workflow_descriptions/{id}/generate_api_key`. **Cannot** be reused across environments.

## Cascades (multi-step flows worth a short name)

- **Auth cascade** — rotating the HMAC requires rolling all three services per env (web-platform, qbrick, backend-ui), or JWTs break.
