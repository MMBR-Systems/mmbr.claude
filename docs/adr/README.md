# ADRs

Architecture Decision Records answering *"why we chose X over Y"*. One file per decision, sequentially numbered `000N-slug.md`.

Offer an ADR only when the decision is **hard to reverse**, **surprising without context**, and **the result of a real trade-off** — see [ADR-FORMAT](../../skills/grill-with-docs/ADR-FORMAT.md) for the format and criteria.

Read by the `pr-review` skill (filtered by blast radius — paths/symbols touched in the diff). Good ADRs prevent reviewers from flagging deliberate architectural choices. Written inline by `grill-with-docs` or via `/preserve`.
