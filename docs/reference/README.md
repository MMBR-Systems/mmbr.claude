# Reference

Catalog-style docs: workspace conventions, repo bookmarks, glossaries, external API contracts. Long-lived, factual content that doesn't fit `architecture/`, `adr/`, or `known-issues/`.

## Conventions

- `workspace-layout.md` — folder convention for the workspace (what `<module>/<service>` means, where personal forks live, etc).
- Other files: catalog-style, named by topic.

Created via `/preserve` (see `.claude/commands/preserve.md`).
