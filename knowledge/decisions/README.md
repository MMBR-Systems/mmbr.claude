# Decisions (ADRs)

Lightweight ADR-style docs answering *"why we chose X over Y"*. One file per decision, named `why-<short-name>.md`.

Auto-loaded by the `pr-review` skill (filtered by blast radius — paths/symbols touched in the diff). Good ADRs prevent reviewers from flagging deliberate architectural choices.

Created via `/preserve` — see `.claude/commands/preserve.md` for the ADR template.
